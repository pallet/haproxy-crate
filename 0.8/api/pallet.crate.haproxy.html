proxied-by

fn

[proxy-group-name app-group {:keys [server-port addr backup check cookie disabled fall id inter fastinter downinter maxqueue minconn port redir rise slowstart source track weight], :as config} & {:keys [instance-id], :as options}]

Declare that a node is proxied by the given haproxy server. This should be called in a phase that runs before :configure (such as :settings).

(proxied-by :haproxy :app1 {:check true}).

settings

fn

[{:keys [instance-id version config], :as settings}]

Build the configuration settings by merging the user supplied ones with the OS-related install settings and the default config settings for HAProxy.

Configuration is via the :config map. :global and :defaults both take maps of keyword value pairs. :listen takes a map where the keys are of the form "name" and contain a :server-address key with a string containing "ip:port", and other keyword/value. Servers for each listen section can be declared with the proxied-by function.
